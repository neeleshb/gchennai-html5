Offline & Storage

Serving without the server.

Web Storage

local storage, session storage.

  • Web pages allowed to store named key/value pairs locally, within the client web browser.

  • Local Storage:  The data persists even after the user closes the browser window.

  • Session Storage:  The data is available only for the particular session, and is deleted when the tab closes.

  • Use cases: Storing user preferences or game states locally.

  • Unlike cookies, the stored data is never sent to the remote server.

  • Implemented natively in the web browsers.

  • 5 MB limit per origin, not-extendable.

Web Storage

Application Cache

what and why?

  • A local cache where the entire pages are cached.
    • Users can browse the websites even when they are offline.
    • The pages load faster (as they come from cache).
    • Server load is reduced: browser will only download changed resources.

  • The server decides whether the page should be cached.
    • manifest file contains list of all resources the browser might need in the offline mode.
    • Every 'cachable' HTML page must have the path of the manifest file set in the 'manifest' attribute of HTML tag.

  • Programmatic access (window.applicationCache) to manipulate the application cache.

Connectivity

Stay connected.

Web Sockets

what and how?

  • API to establish a full-duplex, bidirectional communication over the web.

  • Both the server and client can send data any time, even at the same time.

  • Only the data is sent, not the HTTP headers.

  • Enables cross-domain communication.

  • The websocket protocol (ws://) uses the HTTP upgrade system and "upgrades" an HTTP connection to a websocket connection.

  • Use cases: Multiplayer online games, chat application, sports ticker, etc.

Web Sockets

History API

bring it back!

  • Standardized way to manipulate browser history via scripts.

  • APIs to edit browser entries and change the URL in the location bar.

  • Events to capture the change in the browser's stack that happen on clicking back or forward button.

  • Most common use case: Preserving the navigation without actually reloading the pages:
    • Load only that part of new page that is different from current page, using XMLHttpRequest.
    • Swap in the changed content using DOM manipulation methods.
    • Update the browser location and the browser history using the History API.

History API

history.pushState, history.popState, history.go

function setColor(color) {
    $('bigBox').style.backgroundColor = color;
    var url = window.location.pathname + '#color=' + color;

}

    var color = window.location.hash.split('=')[1] || 'white';
